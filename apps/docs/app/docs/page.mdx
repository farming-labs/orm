# Overview

Farming Labs ORM is for reusable packages that want **one** storage-facing schema, **one** generation story, and **one** documentation home instead of maintaining separate adapters per ORM.

**Quick links**

- [Getting started](/docs/getting-started) — define a schema, add `farm-orm.config.ts`, run `farm-orm generate`.
- [Unified schema](/docs/schema) — models, fields, defaults, references, and relations in one DSL.
- [CLI generation](/docs/cli) — how the CLI loads schemas and writes Prisma, Drizzle, and SQL.
- [Use cases](/docs/use-cases) — auth kits, billing modules, internal platforms.

## At a glance

- **Core** — The schema is the contract shared by runtime, generators, docs, and examples.
- **CLI** — Prisma, Drizzle, and safe SQL outputs come from the same source files.
- **Workspace** — Docs build, tests, generation, and the demo flow stay aligned in one repo.

## What ships today

| Layer                   | Status  | Notes                                                                 |
| ----------------------- | ------- | --------------------------------------------------------------------- |
| `@farming-labs/orm`     | Live    | Schema DSL, manifest, generators, typed helpers, memory runtime       |
| `@farming-labs/orm-cli` | Live    | Loads schemas and generates target artifacts                          |
| Prisma generation       | Live    | PostgreSQL, MySQL, and SQLite providers                               |
| Drizzle generation      | Live    | `pg`, `mysql`, and `sqlite` dialects                                  |
| Safe SQL generation     | Live    | Forward-safe SQL for PostgreSQL, MySQL, and SQLite                    |
| Runtime driver packages | Planned | Prisma, Drizzle, Kysely, SQL, and Mongoose can layer on next          |
