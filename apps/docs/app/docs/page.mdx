# Overview

Farming Labs ORM gives library authors one schema and one query API, while
letting each app choose its own runtime stack.

The model is simple:

1. Define a schema in `@farming-labs/orm`
2. Generate Prisma, Drizzle, or SQL artifacts with `@farming-labs/orm-cli`
3. Run the same typed API through a runtime driver

In practice, that means:

- one schema for shared packages
- one typed query surface for runtime code
- one generator CLI for app-side artifacts
